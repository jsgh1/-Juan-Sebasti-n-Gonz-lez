# 📚 Resumen de la Clase – Actividad 1  

## 📄 Resumen General:  
En esta sesión se estudiaron conceptos clave de **gestión de código en entornos colaborativos**, con énfasis en el uso de **Merge Requests (MR)** y el flujo de trabajo entre distintos ambientes de desarrollo:  
**dev** (Desarrollo), **qa** (Pruebas de calidad), **staging** (Preproducción), **main** (Producción) y **project** (Gestión general).  

También se revisó la **estructura de documentación** para registrar el trabajo realizado, los **tipos de MR**, así como aspectos de **roles y seguridad** en el equipo.  
Finalmente, se asignaron tareas individuales y grupales, incluyendo la elaboración de un **README personal**, la **documentación de la sesión 1** y la **preparación de material** para la socialización del **jueves 14**.  

---

## 📌 Temas Trabajados:  

### 1. Documentación del trabajo realizado  
- Registrar cada paso, acción y resultado.  
- Mantener claridad y detalle en la descripción de procesos.  

### 2. Ambientes de trabajo  
- **dev** → Desarrollo  
- **qa** → Pruebas de calidad  
- **staging** → Preproducción  
- **main** → Producción  
- **project** → Gestión del proyecto  

### 3. Merge Request (MR)  
- Definición, propósito y flujo de aprobación.  

### 4. Tipos de MR  
- **Feature** → Nueva funcionalidad  
- **Bugfix** → Corrección de errores  
- **Hotfix** → Corrección urgente en producción  
- **Refactor** → Mejora de código sin cambiar funcionalidad  
- **Chore** → Tareas de mantenimiento  
- **Spike** → Investigación o prueba técnica  

### 5. Roles y Seguridad  
- Asignación de permisos.  
- Revisión de código antes de aprobación.  
- Control de cambios y trazabilidad.
